# Query Patterns Reference

## Effective Query Patterns

### Factual Extraction
- "What are the key metrics mentioned in [topic]?"
- "List all action items from the meeting notes"
- "What dates and deadlines are referenced?"

### Comparative Analysis
- "How does approach A compare to approach B?"
- "What are the pros and cons of [topic]?"
- "What changed between the Q1 and Q2 reports?"

### Synthesis
- "Summarize the main themes across all sources"
- "What are the three most important takeaways?"
- "What consensus emerges from these documents?"

### Specific Lookups
- "What did [person] say about [topic]?"
- "What is the definition of [term] used here?"
- "What evidence supports [claim]?"

## Cross-Notebook Query Patterns

Cross-notebook queries work best for:
- Finding information when unsure which notebook contains it
- Identifying contradictions or agreements across document sets
- Building a comprehensive view of a topic from multiple sources

Example queries:
- "What do all my notebooks say about pricing strategy?"
- "Find any mentions of [company name] across all notebooks"
- "What recurring themes appear across my research?"

## Research Workflow Patterns

### Augmented Research
1. Query existing notebook for gaps
2. Start web research to fill gaps
3. Import findings
4. Re-query with enriched sources

### Competitive Intelligence
1. Create notebook with competitor docs
2. Run web research for latest news
3. Import and query for analysis
4. Generate report via studio

### Literature Review
1. Add academic papers as sources
2. Query for methodology comparisons
3. Run research for related work
4. Cross-notebook query for synthesis

## Rate Limiting

- Free tier: ~50 queries/day
- Queries count against the daily limit regardless of type
- Cross-notebook queries count as one query
- Research operations count separately from queries
- Space queries throughout the day rather than bursting
